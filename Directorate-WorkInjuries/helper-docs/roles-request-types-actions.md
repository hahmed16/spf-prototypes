# Roles, Request Types, and Available Actions

This document provides a comprehensive mapping of user roles, request types they can access, and the actions available to each role for each request type and status.

---

## Role Overview

The Work Injuries Directorate system includes the following user roles:

### External Roles
1. **Worker (العامل)** - The insured person submitting requests
2. **Employer Delegate (المفوض من جهة العمل)** - Authorized representative from employer
3. **Licensed Institution (المؤسسة الصحية المرخصة)** - Licensed health institution
4. **Hospital Delegate (مندوب المستشفى)** - Hospital representative

### Internal Roles - Investigation
5. **Injury Investigator (موظف قسم التحقيق في إصابات العمل)** - Work injury investigation staff
6. **Injury Head (رئيس قسم التحقيق في إصابات العمل)** - Head of work injury investigation section
7. **Occupational Disease Investigator (موظف قسم التحقيق في الأمراض المهنية)** - Occupational disease investigation staff
8. **Occupational Disease Head (رئيس قسم التحقيق في الأمراض المهنية)** - Head of occupational disease investigation section

### Internal Roles - Sick Leave
9. **Sick Leave Employee (موظف قسم الإجازات المرضية)** - Sick leave section staff
10. **Sick Leave Head (رئيس قسم الإجازات المرضية)** - Head of sick leave section

### Internal Roles - Disability & Chronic Diseases
11. **Disability Employee (موظف قسم الإعاقة والأمراض المستديمة)** - Disability section staff
12. **Disability Head (رئيس قسم الإعاقة والأمراض المستديمة)** - Head of disability section

### Internal Roles - Committees
13. **Committees Employee (موظف قسم اللجان الطبية)** - Medical committees section staff
14. **Committees Head (رئيس قسم اللجان الطبية)** - Head of medical committees section

### Internal Roles - Licensing
15. **Licensing Employee (موظف قسم التراخيص والرقابة)** - Licensing section staff
16. **Licensing Head (رئيس قسم التراخيص والرقابة)** - Head of licensing section

### Committee Roles
17. **Supervisory Committee (لجنة الإشراف)** - Supervisory committee members
18. **Appeals Committee (لجنة التظلمات)** - Appeals committee members

### Rapporteur Roles
19. **Institution Rapporteur (مقرر المؤسسة الصحية المرخصة)** - Licensed institution rapporteur
20. **Appeals Rapporteur (مقرر لجنة التظلمات)** - Appeals committee rapporteur
21. **Supervisory Rapporteur (مقرر لجنة الإشراف)** - Supervisory committee rapporteur

## Action Workflow Patterns

### Investigation Workflow
1. **Investigator**: Check-out → Investigate → Add field visits → Recommend → Check-in
2. **Head**: Review → Approve/Reject/Return → Add notes → Forward to next stage

### Sick Leave Workflow
1. **Employee**: Review → Add periods → Recommend → Forward to head
2. **Head**: Review → Approve periods → Request institution presentation → Monitor disbursement

### Committee Workflow
1. **Employee**: Review → Verify → Recommend → Forward to head
2. **Head**: Review → Schedule session → Monitor session
3. **Rapporteur**: Schedule → Prepare → Monitor → Record → Submit
4. **Committee**: Confirm attendance → Enter decision → Sign electronically

### Licensing Workflow
1. **Employee**: Check-out → Verify → Inspect → Recommend → Check-in
2. **Head**: Review → Schedule session → Monitor → Issue license
3. **Institution**: Submit → Attend session → Receive license

### Appeal Workflow
1. **Employee**: Review → Verify → Recommend → Forward to head
2. **Head**: Review → Schedule session → Monitor session
3. **Rapporteur**: Schedule → Prepare → Monitor → Record → Notify
4. **Committee**: Review → Enter decision → Sign
5. **Appellant**: Submit → Attend session → Receive decision

Available to: Heads

Available to: Worker
Actions: Submit appeal

Available to: Committee members, Rapporteurs

Available to: Committee members

Available to: Committee members

## Notes

1. **Conditional Actions**: Many actions are only available based on specific conditions (e.g., check-out status, request ownership, role permissions).

2. **Sequential Workflows**: Actions often follow specific sequences. For example, an investigator must check-out a request before taking actions on it.

3. **Approval Hierarchies**: Most workflows involve multiple approval levels (employee → head → committee → final decision).

4. **Cross-Role Communication**: Some actions trigger notifications to other roles (e.g., returning a request notifies the recipient).

5. **Audit Trail**: All actions are recorded in the timeline for audit purposes.

6. **Status Transitions**: Actions typically result in status transitions that determine what actions are available next.

7. **Role-Specific Views**: Each role sees different information and has different action options based on their responsibilities.

8. **Permission Levels**: Some roles have view-only access for certain request types, while others have full action capabilities.

9. **Time-Based Actions**: Some actions are time-sensitive (e.g., scheduling sessions, monitoring expiry dates).

10. **System Automation**: Some status transitions and actions are automated by the system based on predefined rules.
